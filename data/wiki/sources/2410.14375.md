## Core Mechanism
Causal Fine-Tuning (CFT) addresses the latent confounded shift problem by using a structural causal model (SCM) as an inductive bias. The method decomposes learned representations into two components: (1) **high-level stable (causal) features** that capture true causal relationships between input and output, and (2) **low-level shift-sensitive (spurious) features** that encode spurious correlations induced by hidden confound variables. The approach derives sufficient identification conditions that motivate a fine-tuning objective for this decomposition. In practice, instantiated with BERT, the method learns both causal and spurious representations simultaneously, then adjusts the predictions to rely more heavily on the stable causal components while down-weighting the shift-sensitive spurious components.

## Key Insight
The key insight is that hidden confounders create spurious correlations between input features and outputs during training, but these correlations break under distribution shift. By explicitly modeling the causal structure through a structural causal model, one can identify which representation components are stable across shifts (causal) versus which are sensitive to the confounder (spurious). The method exploits the intuition that causal features remain invariant while spurious features change under different confounding conditions, allowing the model to generalize better by prioritizing causal information.

## Domain-Blind Abstraction
The core computational pattern is: (1) assume there exists a hidden variable that influences both the input and output, creating misleading statistical associations; (2) learn to separate input representations into components that are genuinely caused by the target versus components that are artifacts of this hidden variable; (3) use only the genuine causal components for prediction. This is essentially a representation decomposition problem where the goal is to isolate signal from noise induced by unobserved confounding.

## Cross-Domain Potential
- **Computer Vision**: Could separate object identity from background/contextual features that spuriously correlate with labels (e.g., recognizing that "snow" often appears with "dog" images due to data collection bias)
- **Medical Diagnosis**: Could distinguish genuine disease markers from hospital-specific artifacts or measurement biases that correlate with diagnoses in training data
- **Financial Modeling**: Could separate true causal economic indicators from spurious correlations induced by market regimes or institutional factors
- **Recommendation Systems**: Could decompose user preference signals from platform-specific biases or exposure effects that create false correlations
