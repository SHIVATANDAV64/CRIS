## Core Mechanism
Template-based Data Generation (TDG) uses frontier LLMs (GPT-4) to automatically generate **parameterized meta-templates** — structured blueprints that encode problem types, variables, and solution logic. These meta-templates can then synthesize a virtually infinite stream of diverse, high-quality problems. Each generated problem includes a programmatically verifiable solution, ensuring quality at scale. The system creates TemplateGSM, a dataset of 7 million grade school math problems with verifiable answers.

## Key Insight
The fundamental bottleneck in training reasoning-capable LLMs is not model architecture but **data scarcity** — the lack of large-scale, high-quality, verifiable training examples. By using an LLM to generate problem templates rather than individual problems, TDG achieves combinatorial diversity: a finite set of meta-templates produces exponentially many distinct problem instances. The programmatic verifiability ensures that generated data is correct by construction, eliminating the quality ceiling of human-annotated datasets.

## Domain-Blind Abstraction
A generative system that uses a capable teacher model to create parameterized problem blueprints, which then produce unlimited problem instances with built-in answer verification. This is a **template-based synthesis** approach: define a parameterized structure once, then instantiate it infinitely many times with different variable bindings. The verification mechanism ensures every output is provably correct.
