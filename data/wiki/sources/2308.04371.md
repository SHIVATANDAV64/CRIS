## Core Mechanism
Cumulative Reasoning (CR) is a structured framework that orchestrates multiple LLMs in three distinct roles to solve complex problems through iterative reasoning:

1. **Proposer**: Generates intermediate reasoning steps (propositions) that decompose the main problem into smaller sub-tasks
2. **Verifier(s)**: Validates each proposed proposition for correctness and consistency before accepting it into the reasoning chain
3. **Reporter**: Composes verified propositions into a coherent final solution

The system builds a **dynamic Directed Acyclic Graph (DAG)** of verified propositions, where each node represents a validated intermediate step. This DAG structure allows for branching reasoning paths, backtracking when errors are detected, and cumulative knowledge building—unlike linear chain-of-thought approaches.

## Key Insight
Human problem-solving is fundamentally iterative and cumulative, not a single-pass feed-forward process. By emulating this human-like thought process with explicit role分工 (role division) and verification at each step, CR prevents error propagation that plagues single-pass LLM reasoning. The verification mechanism acts as a self-correcting loop, ensuring that each intermediate step is sound before building upon it.

## Domain-Blind Abstraction
This is a **multi-agent verification framework** where:
- One agent generates candidate solutions to sub-problems
- Separate validation agents check each candidate for correctness
- A composition agent assembles validated components into the final answer
- All validated components are stored in a graph structure that tracks dependencies

This is essentially a **parallelized, verified iterative refinement** approach to complex task decomposition.
