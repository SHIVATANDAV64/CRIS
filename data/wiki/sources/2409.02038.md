## Core Mechanism
BEAVER is a benchmark dataset comprising 9,128 question-SQL pairs from 812 tables across 19 enterprise domains, derived from real-world private data warehouse query logs. The benchmark addresses two key challenges: (1) scarcity of enterprise query logs due to privacy constraints, and (2) difficulty in error diagnosis with binary accuracy metrics. It synthesizes high-fidelity, expert-verified queries that isolate individual challenges—domain knowledge, query complexity, or their combination. At the evaluation level, it provides human annotations and metrics for five critical subtasks enabling fine-grained analysis of model performance.

## Key Insight
Current text-to-SQL benchmarks use public databases with well-structured schemas and simplistic queries, which fail to capture the complexity of real enterprise environments involving intricate schemas, domain-specific knowledge, and sophisticated analytical queries. This creates a significant gap: LLMs excel on existing benchmarks but struggle with private enterprise data. The key insight is that error diagnosis requires decomposing the text-to-SQL task into subtasks (e.g., schema linking, domain knowledge application, complex query structure generation), as even with SOTA agentic frameworks using GPT-5.2, accuracy reaches only 10.8%—improving to 30.1% with oracle hints for all subtasks.

## Domain-Blind Abstraction
Creating evaluation benchmarks that capture real-world complexity by using authentic data sources (with privacy safeguards) and designing fine-grained evaluation metrics that decompose complex tasks into isolable subtasks. This allows researchers to identify specific bottlenecks in AI systems rather than just measuring end-to-end accuracy.
