## Core Mechanism
The paper proposes a backpropagation algorithm for training spiking neural networks (SNNs) that encodes information using relative multiple spike timing. Unlike conventional timing-based methods (like time-to-first-spike coding) that restrict each neuron to fire at most once, this algorithm allows each neuron to fire multiple times. The key innovation is computing accurate gradients with respect to spike timing in a multi-spike scenario, which enables proper temporal coding while extending the computational capacity of SNNs.

The algorithm maintains the temporal coding advantage of conventional timing-based methods—where information is encoded in the precise timing of spikes rather than just spike counts—but removes the single-spike restriction. This is achieved by modifying the gradient computation to handle multiple spike events per neuron while preserving the differentiable relationship between membrane potential and spike timing.

## Key Insight
The restriction to single-spike firing in conventional timing-based SNNs artificially limits their computational capacity. By allowing neurons to fire multiple times and propagating gradients through all spike events, the network gains additional degrees of freedom for temporal coding. The insight is that multi-spike timing provides richer temporal patterns that can encode more information, and this is particularly influenced by the time constant of postsynaptic currents and membrane potential dynamics—there exists an optimal time constant that maximizes test accuracy, a phenomenon not observed in single-spike-restricted SNNs.

## Domain-Blind Abstraction
This paper addresses the problem of training a neural network where information is encoded in the precise timing of discrete events (spikes), and extends the training algorithm to allow multiple events per processing unit. The core abstraction is: given a network where nodes communicate via discrete timing events, how do we compute accurate gradients when we relax constraints on the number of events per node? The solution involves differentiating through the timing of multiple events while maintaining the temporal order relationships.
