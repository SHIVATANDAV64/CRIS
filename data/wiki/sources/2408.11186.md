## Core Mechanism
The paper proposes a comparison-based algorithm for sequential multi-issue trading between two rationally self-interested agents who exchange resources from finite categories. The key innovation is interpreting accept/reject feedback as **pairwise state comparisons**, which allows the offering agent to iteratively estimate the responding agent's utility gradient without explicit knowledge of their utility function.

The algorithm works as follows: The offering agent proposes trades, and the responding agent either accepts or rejects. Accepted trades must strictly improve both agents' utilities. Rejected offers provide information that **prunes the space of feasible gradient directions** — essentially eliminating directions that would not benefit the responder. This enables systematic refinement toward mutually beneficial trades.

Mathematically, each rejected offer defines a half-space of gradient directions that cannot lead to acceptance, allowing the offering agent to progressively narrow the set of possible beneficial directions through iterative sampling.

## Key Insight
The fundamental insight is that **binary accept/reject feedback contains rich gradient information** when properly interpreted. A rejection doesn't just indicate "no" — it signals that moving in a particular direction would decrease the responder's utility. By treating each response as a comparison between current and proposed allocations, the algorithm can construct a shrinking feasible set of gradient directions that lead to acceptance.

This works because Pareto-improving trades require moving in directions that benefit *both* agents. The algorithm leverages the assumption of greedy rationality: each agent will only accept trades that improve their own utility. Rejected trades therefore carve out regions of the gradient space that are known to be unfavorable.

## Domain-Blind Abstraction
Two parties iteratively exchange resources through sequential proposals. One party (the offerer) cannot observe the other party's preferences directly but receives binary feedback (accept/reject) after each proposal. By interpreting rejection as information that eliminates certain preference directions, the offerer can progressively narrow down what the other party values. The process continues until either a mutually beneficial agreement is found or it can be proven that no improvement is possible.
